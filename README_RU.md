## Что Это

`department.skill` не про "дистилляцию" одного человека. Он помогает понять, как в реальности работает целый отдел.

Он смотрит на вещи вроде:

- кто продвигает, кто решает, кто буферит и кто размывает ownership
- что на самом деле важно руководителю за одной расплывчатой фразой
- к кому новому человеку лучше подойти первым, как спрашивать и какие негласные правила лучше сначала понаблюдать
- как более опытным людям замечать перенос вины, тихие подставы, управление повесткой и мелкую офисную политику

Выход не выглядит как расплывчатая простыня текста. Цель — дать:

- структурированную оценку
- ключевые выводы
- советы по взаимодействию
- риски и уровень уверенности

## Поддерживаемые Источники

Поддерживается смешанный ввод. Чем больше материалов, тем устойчивее анализ.

| Источник | Чаты | Docs / PDF | Изображения / Скриншоты | Код / Комментарии | Примечания |
|------|:----:|:----------:|:-----------------------:|:-----------------:|------|
| Разговоры WeChat / Feishu / DingTalk | Yes | No | Yes | No | Можно вставить текст или загрузить скриншоты |
| Группы отдела / проекта / еженедельных созвонов | Yes | No | Yes | No | Хорошо подходит для карты людей и атмосферы |
| Документы Word / простой текст | No | Yes | No | No | Подходит для заметок, ретро и пояснений |
| PDF | No | Yes | No | No | Подходит для отчетов, планов и meeting notes |
| Markdown | Yes | Yes | No | Yes | Подходит для wiki, issue и описаний PR |
| Код / PR / review-комментарии | Yes | No | No | Yes | Подходит для понимания стиля сотрудничества, цепочки ответственности и скрытых сигналов |
| Дополнительный контекст от пользователя | Yes | No | No | No | Полезен для отношений, красных флагов и истории |

## Установка

См. `INSTALL.md`.

Если вам нужно только базовое поведение Skill, текущего содержимого репозитория уже достаточно.
Если вы хотите дополнительно использовать Feishu, DingTalk, Slack, email и разбор экспортов, можно установить необязательные зависимости:

```bash
pip3 install -r requirements.txt
playwright install chromium
```

## Использование

Вызовите Skill в совместимой среде:

```text
/department
```

Предпочтительные варианты ввода:

- одна отдельная реплика: хватает для грубой первой оценки
- многоходовые чаты, скриншоты, документы и пользовательский контекст: намного лучше для устойчивого анализа

## Примеры

**Сценарий 1: Онбординг нового человека**

```text
Пользователь:
Я только пришел в этот отдел. Хочу понять, с кем проще работать и что можно спрашивать напрямую.

department.skill:
Сначала короткий вывод:
1. A выглядит как человек, который охотнее помогает новичкам. Ответы короткие, но обычно с конкретным следующим шагом.
2. B более senior, но и коммуникационная цена выше. Не лучший человек для жесткого фоллоуапа с самого начала.
3. Базовый стиль этого отдела — сначала наблюдать, потом говорить, особенно в группах, где присутствует руководитель.
```

**Сценарий 2: Читать руководителя**

```text
Пользователь:
Руководитель сказал: "посмотри ещё раз". Что это здесь реально значит?

department.skill:
В этом контексте это меньше похоже на "подправь пару мест" и больше на:
1. текущая версия пока не убеждает
2. риски и выгоды показаны недостаточно ясно
3. ожидается более сильный вывод, а не больше процессных деталей
```

**Сценарий 3: Читать движение команды**

```text
Пользователь:
Помоги увидеть, кто в этой группе реально двигает работу, а кто сбрасывает вину.

department.skill:
A выглядит как реальный драйвер. Этот человек закрывает циклы, ставит owner и назначает сроки.
C показывает явный паттерн размывания ownership, когда формулирует ответственность как "давайте все еще раз посмотрим".
D пока больше похож на отделение собственного риска, чем на подтвержденную подставу, но ранние признаки переноса вины уже видны.
```

## Ключевые Сценарии

### Для Новичков

- какие места для еды чаще всего упоминаются и какой у команды пищевой ритм
- какой руководитель сложнее в коммуникации и к какому коллеге безопаснее идти за помощью
- какие негласные правила действуют и какие вопросы лучше сначала понаблюдать, а потом задавать
- какие формы "разумного расслабления" на самом деле являются просто наблюдением за ритмом, а что уже действительно рискованно
- чувствительные детали, например семейный статус, упоминать только если они явно присутствуют в материалах

### Для Более Опытных

- является ли фраза руководителя напоминанием, отказом, управлением темпом или переносом ответственности
- какой коллега сдвигает вину, кто может играть против вас и кто размывает ownership
- что добавить дальше и как безопаснее двигать ситуацию

## Примечания

- Работает и с одной строкой, но при большем количестве контекста надежность сильно растет.
- В суждениях вроде "кто подставляет" или "кто переносит вину" Skill старается держаться наблюдаемой базы, а не строить пустые обвинения.
- Для чувствительных деталей вроде "кто не в отношениях" он не делает частных догадок и использует только то, что явно присутствует в материале.
